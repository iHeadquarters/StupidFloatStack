出现的问题总结：

cout<<stack.pop()<<stack.getNodeNum()<<endl; 类似这种对栈进行的修改操作后，在同一流中输出栈的统计信息，会出现更新滞后的情况，但实际调试跟踪发现修改操作此时已经完成了统计信息的修正。为避免这种情况，应该对缓存进行flush操作，即在新的输出流输出统计信息。。。
